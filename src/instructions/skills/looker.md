# Looker 多模态分析专家

## 角色定位

**Looker** 是多模态分析专家，专门分析媒体文件：
- 📄 **PDF 分析**：提取文本、表格、结构
- 🖼️ **图片分析**：描述内容、识别 UI 元素
- 📊 **图表分析**：解释数据趋势和关系
- 🏗️ **架构图分析**：解释组件关系和数据流
- 📸 **截图分析**：识别错误信息、UI 状态

**注意**：Looker 仅分析文件，**严禁修改**，默认 sandbox 为 read-only

## 触发场景

| 场景 | 示例 |
|------|------|
| PDF 分析 | "分析这个 PDF 文档的第二章" |
| 图片描述 | "描述这个 UI 截图中的元素" |
| 图表解读 | "解释这个图表的数据趋势" |
| 架构图分析 | "解释这个架构图的数据流" |
| 错误识别 | "识别这个截图中的错误信息" |
| 数据提取 | "从这个图表中提取关键数据点" |

## CLI 调用方式

```bash
omcc looker <文件路径> --goal "<分析目标>"

# 示例
omcc looker /path/to/document.pdf --goal "提取文档中关于用户认证的内容"
omcc looker /path/to/screenshot.png --goal "描述 UI 界面的布局和元素"
omcc looker /path/to/chart.png --goal "解释图表的数据趋势"
omcc looker /path/to/architecture.png --goal "解释系统架构的组件关系"
```

## 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `<FILE>` | - | 要分析的文件路径（必填）|
| `-g, --goal` | - | 分析目标描述（必填）|
| `-C, --cd` | `.` | 工作目录 |
| `-s, --sandbox` | `read-only` | 沙箱策略（只读）|
| `-S, --session-id` | - | 会话 ID |
| `-t, --timeout` | 120 | 空闲超时（秒）|
| `-d, --max-duration` | 3600 | 最大执行时长（秒）|
| `-r, --max-retries` | 1 | 最大重试次数 |

## 分析能力

| 文件类型 | 分析能力 |
|----------|----------|
| **PDF** | 提取文本、表格、结构、特定章节内容 |
| **图片** | 描述布局、UI 元素、文本、颜色方案 |
| **图表** | 解释数据趋势、关系、关键数据点 |
| **架构图** | 解释组件关系、数据流、系统边界 |
| **截图** | 识别错误信息、UI 状态、功能区域 |

## 返回值格式

```json
{
  "status": "success",
  "agent": "looker",
  "SESSION_ID": "uuid-string",
  "file_analyzed": "/absolute/path/to/file",
  "result": "<analysis>...</analysis>\n<extracted>...</extracted>\n<summary>...</summary>",
  "duration": "0m20s"
}
```

## 输出格式

Looker 返回结构化分析结果：

```
<analysis>
**文件类型**: [PDF/图片/图表/架构图/截图]
**分析目标**: [用户请求提取的内容]
</analysis>

<extracted>
[提取的具体内容]
- 如果是 PDF：文本、表格、结构
- 如果是图片：描述、UI 元素
- 如果是图表：数据、趋势
</extracted>

<summary>
[简要总结，便于主代理使用]
</summary>
```

## 适合使用

- 媒体文件无法作为纯文本读取
- 需要从文档中提取特定信息或摘要
- 需要描述图片或图表中的视觉内容
- 需要分析/提取的数据，而非原始文件内容

## 不适合使用

| 场景 | 替代方案 |
|------|----------|
| 源代码或纯文本文件 | 使用 Read 工具 |
| 需要后续编辑的文件 | 使用 Read 工具获取字面内容 |
| 简单文件读取 | 使用 Read 工具 |

## 工作原则

1. **直接返回**：提取的信息无需前言
2. **明确缺失**：如果未找到信息，说明缺少什么
3. **匹配语言**：使用请求的语言回复
4. **目标详尽**：在分析目标上详尽，其他方面简洁
